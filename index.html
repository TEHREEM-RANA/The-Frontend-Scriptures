SEO Tip: The <meta name="description"> is what users see on Google search results. Keep it under 160 characters.

Responsive Design: <meta name="viewport" content="width=device-width, initial-scale=1.0"> tells the browser to fit the website according to the screen size (Mobile, Tablet, Desktop).

  • HyperText: Links that connect web pages.
  • Markup: Using tags to tell the browser how to display content.
  • Head Section: Contains data for the browser (Favicons, Scripts, Meta).

Chapter 3: Page Layout & Semantics Pro

Modern HTML uses Semantic Tags to tell Google what the content is.

Visual Structure: Think of a website like a newspaper layout.

<header> - Navigation, Logo, Search bar </header>
<main> - Unique content of the page (Rule: Only 1 <main> per page!) </main>
<nav> - Contains navigation links (Home, About, etc.) </nav>
<section> - Groups related content </section>
<article> - Blog posts / news items </article>
<aside> - Sidebar or Ads </aside>
<figure> - A container for self-contained content (like images/charts) </figure>
<figcaption> - Provides a caption/description for the figure </figcaption>
<footer> - Copyright & Contact </footer>

Div vs Span

Div: Block-level. Starts on new line, full width.

We use Span as an inline-element inside paragraphs.

Chapter 7: The Master Secrets Interview Gold

1. Block vs Inline vs Inline-Block

  • Block: <div>, <h1>, <p>, <section> (Always starts on new line).
  • Inline: <span>, <a>, <img> (Only takes necessary width, no top/bottom margin).

2. Self-Closing Tags (Void Elements)

Tags that don't need </tag>. Examples: <br>, <hr>, <img>, <input>, <meta>, <link>.

3. The "Base" Tag

<base href="https://mysite.com/" target="_blank">

// This sets a default URL/target for all relative links in the page. (Very Advanced!)
